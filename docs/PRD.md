# 1. 문제 정의
## 타겟 사용자
- 이성 교제를 원하는 사람 중 
- 이성을 만날 기회를 얻고 싶은 사람 중
- 외부 활동을 못하거나 안하는 사람 중
- 데이팅 앱을 안 쓰고 있는 사람 중
- (1) 사회적 인식 때문에 안 쓰는 사람 (2) 신뢰성 때문에 안 쓰는 사람
---
# 2. 해결 방안
## 페인포인트 해결
### 부정적인 사회적 인식
- 모두가 같은 사이클 주기(예를 들어, 일주일)를 갖도록 하여, 대학생들만의 하나의 설레는 이벤트처럼 느껴지게 한다.
- 게이미피케이션 요소를 넣음으로써, 일반적인 매칭 앱과 다르게 좀 더 가볍고 재밌게 느껴지게 한다. 
### 신뢰성
- 권역 기반으로 주변 학교들만 매칭되게 함으로써, "대학생 + 같은 권역" 이라는 동질감을 기반으로 신뢰감을 형성한다.
## 기능 요구사항
### 요약
- 같은 대학교 캠퍼스 내에 있는 사람들끼리 매칭해주는 앱. 
- 매칭 결과는 호감도 수치를 기반으로 연인/썸/친구 중 하나로 결정되며, 이는 평행우주 시뮬레이션 결과로 구분된다. (평행우주 시뮬레이션이란, 1년 동안 유저의 페르소나가 주입된 AI Agent가 가상세계에서 살도록 하는 것)
### user flow
#### 온보딩
1. 약관 동의
2. 기본정보(이름, 성별, 전공, 학번, 학기, 나이) 입력
3. 가치관 이지선다 A/B 질문
4. 관심사 선택
5. AI 페르소나 생성 채팅
6. 휴대폰 SMS 인증
7. 학교 이메일 인증
8. 키 입력
9. 프로필 사진 업로드
10. 닉네임 입력
#### 사이클 흐름
1. D-1 19:00 ~ D1 16:00: 신청 화면(Cycle N)
2. D1 16:00 ~ D1 19:00: 시뮬레이션 진행(Cycle N)
3. D1 19:00 ~ D4 19:00: 시뮬레이션 결과를 3일동안 노출(Cycle N)
4. D4 19:00: 매칭 결과 발표(Cycle N) + 다음 사이클(Cycle N+1) 시작
5. D5 07:00: 프로필 공개(Cycle N)
6. D5 19:00: 사이클 종료(Cycle N)

# 3. 비목표
- 결제/유료화는 하지 않는다.
---
# 4. 성공 지표
1. 자신의 페르소나가 주입된 AI Agent가 자신과 비슷하다고 느낀다.
2. 앱 내 매칭이 되었을 때, 시뮬레이션 결과에서 나타나는 상대방과의 성격 적합성이 실제와 유사하다.
3. 기존의 데이팅앱과 달리, 캠퍼스 기반의 하나의 대학교 이벤트로써 인식한다.
